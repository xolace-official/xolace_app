## Vision

### 10x Check
The monthly awareness card is the seed of a "Moments" infrastructure layer. At 10x:
- Any calendar moment (awareness months, significant dates, seasonal emotional patterns, user anniversaries) triggers contextually relevant content in the app
- Events link directly to actions: crisis resources for crisis-adjacent months, themed reflection prompts, community resonance stats ("3,847 people are sitting with something related to Father's Day today")
- The team schedules events 6 months ahead in the Convex dashboard, with A/B copy variants and per-cohort targeting
- The app stops being a timeless void and starts feeling *present* — aware of the world the user is moving through

## Accepted Scope (added to this plan)

### Data model
- Convex `monthlyEvents` table: `slug` (immutable ID — never rename post-publish), `title`, `body`, `ctaLabel`, `ctaRoute` (optional, validated client-side against allowlist), `sessionPrompt` (optional), `imageUrl` (optional — URL string, hosted on CDN/Expo assets), `startDate` (ISO "YYYY-MM-DD"), `endDate` (ISO "YYYY-MM-DD"), `priority` (1=highest)
- `imageUrl` is a URL string (not Convex file storage — simpler for team to manage via dashboard). Recommended dimensions: 16:9 or square, ≤200KB. Rendered above the title in the card when present; card falls back to text-only when absent.

### Query
- `api.monthlyEvents.getActive` — returns ALL events (no server-side date filter). Client filters in user's local timezone: `new Date(e.startDate) <= today && today <= new Date(e.endDate)`, sorted by priority ascending, tie-broken by startDate ascending (first scheduled wins).

### Seen tracking
- Zustand `seenEventIds: { slug: string; seenAt: number }[]` in TogglesSlice. Must be in BOTH slice type AND `partialize()` call (zustand-partialize-bridge pattern).
- **Marked seen on dismiss or CTA tap** (not on mount). Crash before interaction = event re-shows.
- Pruned on app start: drop entries where `seenAt < Date.now() - 13_months`.

### Display
- `MonthlyEventSheet` — on each app open, filter active+unseen events, show highest priority one. While `useQuery` is undefined (loading/offline): show nothing.
- **Detached bottom sheet** — `detached={true}` goes on `BottomSheet.Content`, not the wrapper:
  ```tsx
  <BottomSheet.Content
    detached={true}
    bottomInset={insets.bottom + 12}
    className="mx-4"
    backgroundClassName="rounded-[32px] bg-overlay"
    handleIndicatorClassName="opacity-0"
    enablePanDownToClose={false}
    accessibilityViewIsModal
  >
  ```
  Floating card, 12px above safe area, 16px horizontal margins, 32px corner radius all sides. No backdrop/blur overlay — the card floats above the live home screen (ambient, not modal). Card shadow: soft ambient warm shadow.

- **Card layout (image present):**
  - Image container: fixed 180px height, full card width, top corners rounded 32px (to match card), `contentFit="cover"`. Upload recommendation: 686×360px (2x for 343px container), JPG/WebP, ≤200KB.
  - Skeleton placeholder from `heroui-native` fills image slot (same 180px height) while loading, disappears on `onLoadEnd` or `onError`.
  - Content area: `paddingHorizontal: 20, paddingTop: 16, paddingBottom: 20`
  - Title: `SpaceGrotesk-SemiBold`, ~20px, `text-foreground`, `accessibilityRole="header"`
  - Body: `Poppins-Regular`, 14px, `text-foreground/80`, `lineHeight: 22`
  - All text: `text-left` (never centered)
  - BottomSheetScrollView wraps title+body (max-height 75% screen height) so long body scrolls

- **Card layout (text-only, no image or image failed):**
  - Same as above without image/skeleton slot; slightly more `paddingTop` (24px) at content start.

- **CTA button (when `ctaLabel` + `ctaRoute` present):**
  - `PressableFeedback`, full-width, `bg-accent rounded-2xl`, height 52px
  - Label: `Poppins-Medium`, 16px, `text-white`, centered
  - `accessibilityLabel={event.ctaLabel}` `accessibilityRole="button"`
  - Pinned below scroll area, not in scroll

- **Dismiss action (always present, below CTA):**
  - `PressableFeedback` (no background), wrapping `AppText`
  - Label: "Not right now", `Poppins-Regular`, 13px, `text-foreground/50`
  - When no CTA: label "I see this", `text-foreground/70` (slightly more prominent)
  - Centered within its own row; `paddingVertical: 12`
  - `accessibilityLabel="Dismiss awareness event"` `accessibilityRole="button"`

- **Image `accessibilityLabel`:** `{event.title} — awareness event image`

- **Copy framing guidance (for team when creating events in Convex dashboard):**
  - Body must NOT use imperative framing ("Join us", "Learn about...", "This month we...")
  - Structure: acknowledgment sentence first (what the user might be feeling), context second, CTA optional invitation
  - No emoji in title or body (enforced by character guidance, not code)

- Kill-switch: set `endDate` to yesterday to cancel a live event.
- `ctaRoute` client-side allowlist validation before `router.push`.

### Analytics
- PostHog: `awareness_event_shown`, `awareness_event_dismissed`, `awareness_event_cta_tapped`

### Session prompt
- `pendingEventPrompt: { text: string; expiresAt: number } | null` in Zustand (expiresAt = seenAt + 7 days). Set on event dismiss if `sessionPrompt` is present.
- On reflect idle mount: if set and not expired, use as prompt and clear. If expired, clear without showing.
