## Current session-end reality (what we integrate into)

Phase-sequential, full-screen, per variant:

```
ExitVariant:      acknowledge ("Heard.")              → close ("Have more?" + Done)
ActivityVariant:  acknowledge ("showed up") → mood    → offer ("Leave it for someone",
                                              (4 pills)  anonymous, distilledText)   → close
```

`SessionEndNotifNudge` and `ReachFeedbackCard` are absolute-positioned siblings, not
cards in a list. The only Bridge breadcrumb in code is `activity-variant.tsx:259`:
`TODO(bridge): replace or extend this phase with BridgeOffer`. That TODO is **not**
the chosen entry — see D1.

## Integration decisions

| # | Decision | Choice |
|---|----------|--------|
| D1 | Entry point | **Close phase** of session-end (the "Have more? I'm here." moment), as a recommended card with a "new" marker — NOT the offer phase. The offer phase is *anonymous strangers*; the Bridge is *a specific person*. Keep them distinct. |
| D2 | First-run experience | Full-screen intro modal on first navigation: what the Bridge is + 3 privacy promises. Seen once, gated by `bridgeIntroSeen` (persisted `toggles` slice, same pattern as `introSeen`). |
| D3 | Screen 1 structure | Single screen (name field + optional relationship pills). Whole feature is one route `/trusted-bridge` with local `step: "recipient" \| "draft"` state — no nested routes. |
| D4 | Generation trigger | **Auto-fire** the Anthropic call when the user taps "Write this together" on screen 1. No separate "Generate" button. Draft is "becoming" as they arrive on screen 2. |
| D5 | Loading state | Full-screen **rotating shimmer text** on screen 2 (see Visuals); draft **fades in** via `EaseView` (~500ms) when it lands. Error → same fade resolves to Mirror-text fallback + "couldn't personalize" note. NO orb/ember. |
| D6 | Variant eligibility | Bridge card appears in the close phase of **both** exit and activity variants, guarded on `mirrorText != null`. An exit session can carry a confirmed Mirror. |
| D7 | Screen 1 exit | Top-left **✕** on screen 1 → `router.replace('/')` (home) + `bridge_dismissed { step: "recipient" }`. Not a back arrow — the close phase is gone after the push. |
| D8 | Analytics | `bridge_shared` payload = `{ character_delta, recipient_relationship }`. **Drop `first_sentence_changed`** — its only purpose was a Stage-1-vs-Stage-2 comparison, and Stage 1 is gone. Re-add later only if opening-quality prompt tuning becomes a question. |

## Wireframes

### Close phase — Bridge entry card (both variants, `mirrorText != null`)
- Card above the existing "Have more?" button, entering with the close phase's `EaseView`
  fade, delayed slightly so it doesn't compete on entry.
- `bg-surface`, `rounded-2xl`, `border border-accent/20`; "· new" accent dot top-right.
- Headline `font-serif text-xl` ("Tell someone" / "Find the words for someone who matters.");
  one-line `text-foreground/50` subtext; single ember glyph as the only icon.
- Tap → `router.push('/(protected)/trusted-bridge', { sessionId, mirrorText })`.

### First-run intro (full-screen, once)
- **Visual (to generate) — "sending" pose.** Image-gen prompt:
  > A cute 3D-rendered mascot character named Flux: a glossy soft-bodied flame/teardrop
  > shape with a smooth curled-over pointed tip at the top, like a friendly little flame.
  > Pastel rainbow gradient body — warm yellow and soft blue at the crown blending down
  > through peach on the face into a smooth blue-purple-pink gradient on the body and stubby
  > legs. Simple peach-orange stubby arms. Two shiny black oval eyes with small white
  > highlights, a tiny gentle closed smile, calm and tender expression. The character is
  > gently releasing a small folded white paper note (or little paper boat) from both hands,
  > letting it float upward; the note drifts up and away trailing a soft warm glowing light
  > trail. Flux is looking up after the note, peaceful and a little wistful. Glossy clay/3D
  > render, soft studio lighting, subtle soft shadows, smooth matte-glossy surface. Centered,
  > full body, generous empty space above for the note to drift into. Transparent background,
  > PNG. Same character design and rendering style as a soft 3D emoji mascot.
  - Match existing `jump-love-bgremove.png` (same character, transparent bg).
  - Optional later motion: Reanimated float-up + fade loop on the note layer.
- `font-serif text-3xl` headline. Three quiet privacy lines (NOT a feature grid): "Stays on
  your device" / "We never see who" / "Nothing sends on its own". One "Begin" button → sets
  `bridgeIntroSeen = true`.

### Screen 1 exit affordance
- Top-left **✕** (close), NOT a back arrow. Tap → `router.replace('/')` + PostHog
  `bridge_dismissed { step: "recipient" }`. Back-swipe stays enabled on this step.

### Screen 1 — recipient
- Header "Who would you like to tell?" (`font-serif text-3xl`).
- heroui-native `TextField`, placeholder "their name or how you think of them".
- 4 relationship pills (parent/partner/friend/sibling), optional, single-select.
  **Pills over a Select** because: 4 fixed/short options is "show them all" territory
  (≤5–6 → inline); one tap vs Select's open→pick→dismiss; no hidden state behind a label
  (a closed Select reads as "must fill," wrong for an optional field); doesn't fight the
  name-field keyboard (no popover/sheet); matches the app's existing mood-pill + texture-tag
  vocabulary; a dropdown reads as a transactional form control, off-tone for this screen.
  Flip to a Select only if the list grows past ~6 or becomes unbounded. **Layout:** compact
  horizontal wrapping row (like texture tags), NOT the mood pills' full-width stacked style;
  selected `border-accent/40 bg-accent/10`; tapping the selected pill again deselects (it's optional).
- "Write this together" CTA disabled until `name.trim() !== ""`; on tap → PostHog
  `bridge_opened` → advance to `draft` → fire AI call.

### Screen 2 — loading → draft
- Loading (**rotating shimmer text**, NO orb/ember): slow cross-fade between 2 short lines
  (~2–2.5s hold each), one gentle L→R shimmer sweep per line (Skia gradient mask over text,
  ~1.5s). No fast ticker, no typing/cursor effect. AI call is ~2–4s so users see 1–2 lines —
  the first line must stand alone. Default pair:
  - "Finding the words for {name}…"
  - "Turning what you felt into something you can say…"
  - Alternates: "Shaping this for {name}…" / "Looking for how to begin…" / "Making this easier to send…"
  - Optional small Flux below in a "composing" pose. Image-gen prompt:
    > A cute 3D-rendered mascot character named Flux: a glossy soft-bodied flame/teardrop
    > shape with a smooth curled-over pointed tip at the top. Pastel rainbow gradient — warm
    > yellow and soft blue at the crown blending through peach on the face into a blue-purple-pink
    > gradient body and stubby legs, with peach-orange stubby arms. Two shiny black oval eyes
    > with small white highlights, a small thoughtful smile. The head is tilted slightly to one
    > side and one little hand is raised near the chin in a gentle thinking/listening pose, eyes
    > looking softly upward as if finding the right words — calm, attentive, caring. Glossy
    > clay/3D render, soft studio lighting, subtle soft shadows. Centered, full body. Transparent
    > background, PNG. Same character design and rendering style as a soft 3D emoji mascot.
    - Match existing `flux-look-mini-bg.png` (same character, transparent bg).
- Draft: `← Back` header (to `recipient`), `font-serif` "Here's a start...", editable
  heroui-native `TextArea` pre-filled with AI draft, "Share..." CTA (`Share.open` in
  try/catch), "Not right now" (→ `router.replace('/')`, `bridge_dismissed`), footer
  "This only lives on your device. We don't see who you're writing to."
- `gestureEnabled: false` on the `draft` step only (toggle via `navigation.setOptions`
  in a `step` effect; `useNavigation` from `@react-navigation/native`).

## Design gaps to handle during build

1. **Keyboard handling** — screen 1 (`TextField`) and screen 2 (`TextArea`) rely on the
   `KeyboardProvider`/`KeyboardAvoidingView` already in `RootProvider`; the primary CTA
   must stay reachable above the keyboard on both.
2. **Accessibility** — relationship pills + Bridge card need `accessibilityLabel` +
   `accessibilityRole`; the "new" dot must not be the only novelty signal for screen
   readers; pills ≥ 44px touch targets; screen-1 ✕ needs a labelled hit target.

## Pre-steps (independent of visuals)

1. Extend `useSessionEnd` to expose `mirrorText: session?.mirrorText ?? null` (never landed;
   hook currently returns `distilledText` only). Pass through `SessionEndScreen` → both variants.
2. Add `bridgeEnabled: boolean` (default `true`) and `bridgeIntroSeen: boolean` (default
   `false`) to the persisted Zustand `toggles` slice.
3. Settings opt-out toggle reading `bridgeEnabled`.

## Carried forward unchanged from CEO plan (Stage-2 Direct)

- `convex/ai/bridge.ts`: `verifyOwnership` internalQuery (wraps `requireSessionOwnership`),
  `_generateDraft` helper (single Anthropic call, no action-from-action), `requestBridgeDraft`
  public action.
- `use-bridge-draft.ts` hook (`idle | loading | success | error`, draft fallback on error).
- recipientName / recipientRelationship used in prompt only, never persisted.
- mirrorText > 500 chars → pass `sessionId` only, query via `api.sessions.getById`.

## Not in scope

- Contact picker, draft persistence ("unsent letters"), push follow-up, deep-link landing.
- Offer-phase changes — the anonymous "Leave it for someone" phase is untouched; the
  `TODO(bridge)` at `activity-variant.tsx:259` should be removed (entry is the close card, not here).
