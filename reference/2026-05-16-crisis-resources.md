## Vision

### 10x Check
The 10x version of this feature is crisis infrastructure, not a resources page.
Xolace becomes the app that a university counselling department trusts enough to recommend to students. The crisis screen has:
- Verified, current hotlines for 150+ countries — maintained by a community of verified contributors
- University counselling section: universities register their services directly
- Proactive surfacing: if a user has shown repeated distress signals across sessions, a notification surfaces the crisis button before they open the app in a bad state
- Resource ratings: users anonymously mark "this helped" or "number was wrong," feeding back into quality scoring
- Open-source resource registry: the data lives in a public GitHub repo anyone can PR to

### Dream State (12-month)
```
CURRENT STATE               THIS PLAN                    12-MONTH IDEAL
No crisis button            Crisis button on idle         Crisis infrastructure layer
AI-triggered escalation     Standalone crisis screen       Proactive crisis surfacing
Ghana-only resources        5-country static JSON          150+ countries, community-maintained
No university support       No university support          University counselling registry
```

## Accepted Scope (in this plan)

### Crisis button wiring
- Add `onCrisisTap: () => void` to `IdleState` Props type in `idle-state.tsx`
- In `idle-state.tsx`, wrap `<QuietReturnHeader>` and the crisis button in a `flex-row justify-between items-start`. `QuietReturnHeader` gets `className="flex-1"` so the button sits flush right without any absolute positioning.
  ```tsx
  <View className="flex-row items-start justify-between pt-10 pb-4">
    <QuietReturnHeader ... className="flex-1" />
    <CrisisButton onPress={onCrisisTap} />
  </View>
  ```
  Remove the standalone `pt-10 pb-4` from `QuietReturnHeader` since the wrapper View now owns that padding.
- Wire `onCrisisTap` in `reflect-screen.tsx` (where `IdleState` is rendered in `renderScreen('idle')`):
  ```tsx
  const router = useRouter(); // add this
  // in renderScreen case 'idle':
  <IdleState ... onCrisisTap={() => router.push('/crisis-resources?from=idle_button')} />
  ```
- Crisis button visual: translucent warning pill — `bg-warning/10 border border-warning/20 rounded-full px-3 py-1.5 flex-row items-center gap-1.5`. Lifebuoy SVG icon (16px) + **"Help"** label (`AppText` `text-xs text-warning`). Rendered inside `PressableFeedback` from `heroui-native`. `accessibilityLabel="Open crisis resources"`. Quiet, not alarming — it's a door. Label is "Help" not "Crisis" to lower the activation threshold (users who feel they're "not bad enough for crisis" will still tap "Help").

### Feature directory `src/features/crisis-resources/`
- `components/crisis-resources-screen.tsx` — main UI (follow `src/features/peer-reflection/components/screen/` colocation pattern)
- `data.ts` — `COUNTRY_RESOURCES: Record<CountryCode, CountryData>`; single module-level `LAST_UPDATED = '2026-05-16'` (not per-country); resources stored **pre-sorted by priority** (ascending) — add comment: `// Resources are in display order; priority field documents the rank. Keep sorted when adding new entries.`
- `use-crisis-resources.ts` — returns `{ selectedCountry, setCountry, countryData }`. Locale detection with null safety. Hook signature: `selectedCountry: CountryCode | null` to support empty state when locale is unrecognized:
  ```ts
  const { from = 'idle_button' } = useLocalSearchParams<{ from?: string }>();
  const region = Localization.getLocales()[0]?.regionCode;
  const [selectedCountry, setCountry] = useState<CountryCode | null>(
    region && COUNTRY_RESOURCES[region as CountryCode] ? (region as CountryCode) : null
  );
  // fire posthog on mount
  posthog.capture('crisis_resources_opened', { country: selectedCountry ?? 'unrecognized', entry_point: from });
  ```
  When `selectedCountry` is null, the screen renders an empty state: "Tap to select your country" instead of the emergency call button. Country selector is always visible so users can pick. Once a country is selected, the emergency button appears.
- `types.ts` — copy (NOT re-export from convex/) the `Resource` type here to avoid frontend→backend import. Also delete the redundant local `Resource` type in `escalation-state.tsx` and import from this file. The remaining `Pressable` buttons in `escalation-state.tsx` (not the ResourceItem) can stay as-is — fixing those is out of scope for this PR.

### Screen components

#### Design Visual Spec (from /plan-design-review, 2026-05-16)

**Typography & spacing:**
- Subtitle (first body element below native header): `text-sm text-foreground/50 pb-6` — "If you need support right now, you're in the right place."
- Section spacing: `mt-6` between subtitle/emergency block and country section; `mt-5` between country section and resource list

**Emergency call button (full visual spec):**
```tsx
<PressableFeedback
  onPress={handleEmergencyCall}
  accessibilityRole="button"
  accessibilityLabel={`Call emergency services — ${emergencyNumber}`}
  accessibilityHint="Opens your phone dialer"
>
  <View className="w-full rounded-2xl border border-warning/30 bg-warning/10 px-5 py-5 flex-row items-center gap-3">
    {/* Phone icon 20px, warning color */}
    <PhoneIcon size={20} className="text-warning" />
    <View className="flex-1">
      <AppText className="text-xs text-warning/70 uppercase tracking-wider">Emergency</AppText>
      <AppText className="text-xl font-medium text-warning">{emergencyNumber}</AppText>
    </View>
    <AppText className="text-xs text-warning/50">Call →</AppText>
  </View>
</PressableFeedback>
```
Rationale: larger than resource cards (`py-5` vs `py-3.5`), `rounded-2xl` vs `rounded-xl` for visual hierarchy. Same warning palette as escalation-state — quiet, not alarming. Phone icon provides semantic clarity.

**Empty state (unrecognized locale — selectedCountry is null):**
```tsx
<AppText className="text-sm text-foreground/40 pb-6">
  Select your country below to see emergency services.
</AppText>
```
Replaces the emergency button. No placeholder card, no illustration — quiet inline text consistent with Xolace voice.

**Country change animation:** `setCountry(newCode)` triggers re-render. `EaseView initialAnimate={{ opacity: 0, translateY: 20 }}` on resource items provides natural stagger fade-in for new country's resources. Old items cut away (acceptable). No additional animation state machine needed.

**Screen entrance animation:** Wrap subtitle + emergency button block in a single `EaseView`:
```tsx
<EaseView
  initialAnimate={{ opacity: 0, translateY: 10 }}
  animate={{ opacity: 1, translateY: 0 }}
  transition={{ type: 'timing', duration: 500, delay: 100, easing: [0.455, 0.03, 0.515, 0.955] }}
>
  {/* subtitle + emergency button */}
</EaseView>
```
Resource list items use existing per-index delay stagger. Header arrives via native navigation (instant).

**Night mode:** Inherit active theme via CSS variables — zero extra code. Manual contrast verification needed: amber warning on dark night-mode background. Add to implementation gates.

**ResourceItem touch targets:** Add `hitSlop={8}` to `PressableFeedback` in `resource-item.tsx` for tappable items (phone/url/email). Brings effective touch target to 44px+ without changing visual height. Non-tappable (text) items have no `hitSlop`.

## Error & Rescue Spec

```
CODEPATH               | FAILURE                    | RESCUED? | USER SEES
-----------------------|----------------------------|----------|--------------------
Linking.openURL(tel:)  | No phone dialer            | YES      | toast: "Phone calls not supported on this device"
Linking.openURL(mailto)| No mail app                | YES      | toast: "No mail app found"
getLocales()[0]        | Returns undefined           | YES      | Empty state: "Tap to select your country"
getLocales()[0]        | Unrecognized region         | YES      | Empty state: "Tap to select your country"
Select (country picker)| Version incompatibility    | N/A      | VERIFIED — heroui-native v1.0.0 supports presentation="bottom-sheet"
```
Use `useToast()` from `heroui-native` (already in codebase). Check `Linking.canOpenURL()` before every `Linking.openURL()` call.
Add `tel` and `mailto` to `LSApplicationQueriesSchemes` in `app.config.ts` to guarantee `canOpenURL` works on all iOS versions:
```ts

```

## Tests

`src/features/crisis-resources/crisis-resources.test.ts` — bun:test, no native mocks needed:
```ts
import { describe, it, expect } from 'bun:test';
import { COUNTRY_RESOURCES } from './data';

describe('country resolution', () => {
  it('resolves GH from locale', () => expect(COUNTRY_RESOURCES['GH']).toBeDefined());
  it('returns null for unrecognized locale (JP)', () => {
    const region = 'JP';
    expect(COUNTRY_RESOURCES[region as keyof typeof COUNTRY_RESOURCES]).toBeUndefined();
  });
  it('all countries have emergencyNumber', () => {
    Object.values(COUNTRY_RESOURCES).forEach(c => expect(c.emergencyNumber).toBeTruthy());
  });
  it('emergencyNumber not duplicated in resources[]', () => {
    Object.values(COUNTRY_RESOURCES).forEach(c => {
      const hasEmergency = c.resources.some(r => r.value === c.emergencyNumber && r.type === 'phone');
      expect(hasEmergency).toBe(false);
    });
  });
  it('resources are pre-sorted (priority ascending)', () => {
    Object.values(COUNTRY_RESOURCES).forEach(c => {
      for (let i = 1; i < c.resources.length; i++) {
        expect(c.resources[i].priority).toBeGreaterThanOrEqual(c.resources[i-1].priority);
      }
    });
  });
});
```

### Data Structure
```typescript
// src/features/crisis-resources/types.ts — copied from convex/ai/safeguard.ts, not imported
// Keep in sync with convex/ai/safeguard.ts Resource type manually
export type Resource = {
  type: 'phone' | 'url' | 'text' | 'email';
  source: 'crisis_line' | 'xolace_support' | 'text_support' | 'local_service' | 'online_resource';
  priority: number;
  label: string;
  value: string;
  description?: string;
};

type CountryCode = 'GH' | 'US' | 'GB' | 'AU' | 'CA';

type CountryData = {
  name: string;
  flag: string;           // emoji flag e.g. '🇬🇭'
  emergencyNumber: string; // shown in top button ONLY — must not appear in resources[]
  resources: Resource[];  // full list shown below the emergency button
};

// Module-level (not per-country):
const LAST_UPDATED = '2026-05-16';
```
