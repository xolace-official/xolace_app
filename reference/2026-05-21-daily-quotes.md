# CEO Plan: Daily Quotes Feature
Generated by /plan-ceo-review on 2026-05-21
Branch: feat(❝)/daily-quotes | Mode: SELECTIVE EXPANSION
Repo: xolace-official/xolace_app

## Problem Statement

Xolace users on the WhatsApp channel are actively sharing quotes posted there to their statuses.
The behavior is real and proven. The goal is to bring that loop into the app: daily quote delivery
with branded sharing cards that drive organic growth (WhatsApp status → App Store install).

A secondary goal: session-derived quotes that are unique to Xolace — not other people's words, but
the user's own emotional context distilled into one shareable sentence. No other app does this.

## Approach Chosen: B — Preference-Curated Free + Session-Derived (moves to premium later)

Two content pipelines:
1. **Preference-curated** (free, always on): User sets 2-4 theme preferences (e.g., "resilience",
   "self-compassion", "relationships"). A quote library serves a new quote daily matching those
   preferences. Library must be large enough for 6+ months without repeats (~180+ quotes minimum).
   Preference questions must feel like texture-word taps (Xolace idiom), not a clinical survey.
   If a user hasn't set preferences yet, serve from the full library (no filter) until they do.

2. **Session-derived** (free now, premium later): Nightly cron generates 1 quote per active user
   from the emotional context of their last 1-3 sessions (up to 3 most recent sessions within the
   last 7 days). Not a direct quote of their words — a distillation of the emotional theme and
   register. Quote is ready when they open the app.
   If no session in 7 days, falls back to preference-curated only. After 1 day without a new
   session, a gentle nudge appears inline on the quotes screen: "Try a session to get a quote
   that meets you where you are today." Dismissed by tapping it or completing a session.

Both are shareable as branded Xolace cards.

## Accepted Scope (in this PR)

- Preference-curated quote pipeline (Convex table + quote library + daily assignment cron)
- Preference setup UX (texture-tag style, 2-4 questions, not a form)
- Session-derived quote pipeline (nightly cron, Anthropic call, last 1-3 sessions as context)
- Daily quotes screen (shows ONE quote: session-derived if available, curated otherwise. Design decision D2: one quote at a time — more intimate, full attention. No two-card layout.)
- Gentle nudge when no session-derived available ("Try a session for a more personal quote") — inline text below curated quote, not a separate card. Bridges curated → session-derived path.
- Quote reaction component (resonates / not today — stored as `reaction` field on `dailyQuotes`)
- Sharing card component (branded PNG/image + two explicit store URLs: iOS App Store + Play Store)
- Opt-in daily notification (Expo Notifications, user sets time, Xolace-voice copy, deep-links to quotes screen)
- Future-premium flag: `isPremium: false` boolean on `dailyQuotes` records (stored, not enforced until subscription ships)
- Quote library content production + Convex seed script (180+ quotes, themed, Xolace-voice)

## New/Modified Convex Tables

- `quotes` — NEW curated library (text, themes[], source, language)
- `dailyQuotes` — NEW daily log (emotionalProfileId, date, type: "session"|"curated", text, sessionContextIds, isPremium: bool, reaction: "resonates"|"not_today"|null)
- `preferences` — EXTENDED (add `quotes?: { themes: string[], notificationEnabled: bool, notificationTime?: string, shownQuoteIds: Id<"quotes">[] }` field)

Note: `reaction` is a field on `dailyQuotes`, NOT a separate table.
Note (D1 — eng review): Use `emotionalProfileId: v.id("emotional_profiles")` throughout, NOT `userId`. Matches existing privacy architecture — emotional data (themes, daily quotes) belongs in the pseudonymous identity layer, not the auth identity layer. All existing emotional tables use `emotionalProfileId`; mixing would break privacy model and add unnecessary join overhead.
Note (D3 — eng review): NO separate `quotePreferences` table. Instead, extend the existing `preferences` table with a `quotes` optional object field. Single source of truth for all notification state — quotes cron must check `preferences.notifications.enabled` (master flag) AND `preferences.quotes.notificationEnabled` (quotes-specific flag). Both must be true to send a quote notification. Use `v.optional(v.object({...}))` so existing rows don't need migration.

## Key Technical Decisions

- **Navigation entry points (both):**
  1. **Idle screen quote pill** — LEFT-ALIGNED with subtle `→` arrow (Design D3). Position: between the Separator and the main `flex-1` writing area in `idle-state.tsx`. Shows first 6-8 words of today's quote + "… →". Tapping navigates to the quotes screen. Discoverability layer for users who never open the menu. Left-aligned is consistent with the existing idle-state layout; the arrow signals navigation without needing a centered layout.
  2. **Idle menu entry** — add a 3rd item to `menu-buttons.tsx` with label **"Today"** (Design D12: not "Quotes" — "Today" matches the daily-refresh rhythm and Xolace's present-tense ethos). Icon: `quote.bubble` (iOS SF Symbol) / `format_quote` (Android). Use `index={2}` for the `AnimatedRow` stagger (per existing pattern: each row needs a unique stagger index). Update `numberOfRows` to 3.

- **QuotesScreen layout (Design D2):** ONE quote at a time. Session-derived if available, curated otherwise. No two-card layout — one quote gets full attention. When showing curated: small inline nudge text below ("Try a session for a more personal quote") — not a card, just subtle text. When session-derived is available: no nudge.

- **PreferenceSetupSheet first-visit (Design D4):** Full-screen on first visit. The setup takes the whole screen — no empty quote state visible behind it. After completing preferences, cold-start generator runs and first curated quote fades in. The transition is: setup → loading state → quote card fade-in (EaseView animation).

- **Interaction states (Design D5, D6, D7):**
  - **Loading** (cold-start, preference setup → first quote): SkeletonGroup from heroui-native renders a pulsing card placeholder while cold-start generator runs (~1-2s). No blank dark screen.
  - **Reaction selected**: Tapped reaction pill shows `text-accent` on the label (same pattern as selected TagGroup items in idle-state). Both "resonates" and "not today" use the same accent color when selected — no weighting between them. The OTHER reaction pill dims to `foreground/20` to indicate the choice is made.
  - **Notification permission denied**: Setup completes normally. After denial, a single line of AppText (`text-xs foreground/40`) appears: "You can enable notifications in Settings anytime." No error state, no re-prompt. The notification time chosen is saved in `preferences.quotes.notificationTime`; `notificationEnabled` remains false until permission granted in Settings.
  - **Error (cold-start generator fails, or network error)**: Quiet inline error below the loading skeleton: "Something went wrong. Pull to refresh." Use existing error-state visual language (foreground/40, no dramatic error color). Do NOT use a full-screen error state for what is a secondary screen.
  - **SharingCard capture loading**: `react-native-view-shot` `capture()` is near-instant but async. Disable the share button with opacity 0.5 and show a spinner icon (SymbolView `arrow.2.circlepath`) while capturing. Re-enable on success or error.

- **Sharing card format:** PNG via `react-native-view-shot`. The project already uses dev builds (native modules like `expo-sqlite/kv-store` are in use; EAS is configured) so this is feasible. Decision: use `react-native-view-shot`. Do NOT use Expo Go during sharing card development.

- **SharingCard visual spec (Design D10):** Background: `--background` CSS variable (warm dark, NOT plain black — uses the app's existing background, which has warmth). Layout: quote text centered vertically (it's the only element). Bottom strip: Xolace wordmark text (not icon logo) bottom-left, iOS App Store + Google Play Store text links bottom-right. Visual depth: `LinearGradient` from transparent to `accent/10` at the bottom, behind the wordmark/links strip. No blobs, no borders, no decorative elements. Typography: Poppins Semibold for the quote (large, 28px on the card canvas), Poppins Regular 12px for the wordmark and links. The card should pass the "would someone screenshot this?" test because of the typographic weight and the warmth of the background.

- **PreferenceSetupSheet step 2 pattern (Design D11):** Notification time options (Morning, Afternoon, Evening, "I'll check myself") use **TagGroup single-select** (`selectionMode='single'`) — same pill tag idiom as step 1. Consistent visual language across both steps. TagGroup already supports single-select mode, no new components needed.

- **App Store links:** Two explicit hardcoded URLs — iOS App Store + Google Play Store. No Branch.io or third-party SDK for this PR. Add attribution tracking later if needed.

- **Minimum session count:** 1 session is enough for session-derived quotes. Use up to 3 most recent sessions within the last 7 days. If user has 4+ sessions in 7 days, use the 3 most recent only.

- **Theme slugs (preference tags):** `resilience`, `self-compassion`, `relationships`, `grief-and-loss`, `change`, `anxiety`, `identity`, `loneliness`. Stored as `string[]` in `preferences.quotes.themes`. Each quote in the library is tagged with 1-2 slugs. User selects 2-4 during preference setup.

- **No-preferences fallback:** If `preferences.quotes` is undefined (user hasn't visited the quotes screen yet), serve from the full unfiltered quote pool.

- **Cron "active user" definition:** A user qualifies for nightly generation if `preferences.quotes` is defined OR at least 1 session in the last 30 days. Do NOT scan all users. Query via `emotional_profiles` paginated, check `preferences.by_profile` for each.

- **Cron scalability:** The nightly cron must NOT call Anthropic for all users in a single action. Pattern: cron calls `internal.quotesDistiller.generateForNextBatch({cursor: null})` → `.paginate({numItems: 20, cursor})` → dispatches `ctx.scheduler.runAfter(0, internal.quotesDistiller.generateForUser, {emotionalProfileId})` per user → if `!isDone`, schedules itself with `{cursor: continueCursor}`. Loop terminates when `isDone === true`. Single file, V8 runtime — no `"use node"` needed (Anthropic SDK uses fetch, not Node.js built-ins; verified by checking existing AI callers in this codebase). Add `isProd` guard at handler start: if not production, log and return early. Consistent with all existing crons.

- **Anthropic failure in cron:** If the Anthropic call fails for a specific user, log the error with `emotionalProfileId` and error type, skip session-derived for that user today (preference-curated still shows). Do NOT retry — the nightly cron runs again tomorrow. Silent failures are not acceptable.

- **dailyQuotes.date format (D7 — eng review):** Always use UTC ISO date string: `new Date().toISOString().split('T')[0]` → `"YYYY-MM-DD"`. The cron, cold-start generation, idempotency check, and client query must all use the same format. Ghana = UTC+0, so UTC date is correct for MVP. Users in other timezones may see "tomorrow's quote" starting at their local midnight if it's already UTC midnight — acceptable for MVP. Add timezone-aware date generation to TODOS.md for global expansion.

- **Content production sequencing (D6 outside voice):** Quote library content (180+ quotes) MUST be produced and seeded BEFORE the cron runs in production. Sequencing: (1) produce content offline → (2) human review → (3) run seed script on prod deployment → (4) enable cron. If the library is empty, the cold-start generator and cron will serve nothing for curated quotes. Content production is a parallel-track deliverable, not a post-ship task.

- **Quote library size:** 180+ quotes. Tonally consistent with Xolace — emotional honesty > positivity. Not hustle-culture, not clinical affirmations. Curated, not scraped. Content production + Convex seed script are explicit deliverables for this PR.

- **Preference onboarding:** Inline card on first visit to the quotes screen. Not in main onboarding. The preference selection uses the texture-tag idiom (8 theme slugs shown as tags, user selects 2-4). Appears before the first quote renders.

- **QuoteCard visual spec (Design D5/Design System):**
  - Quote text: `text-2xl font-semibold` (Poppins Semibold, ~24px). Left-aligned, NOT centered — consistent with app text alignment; "centered everything" is an AI slop pattern.
  - Label ("Your quote today" / "For you today"): `text-xs font-normal text-foreground/40`. Left-aligned.
  - First-ever session-derived quote: label reads "From your sessions — for the first time" (Design D8: content does the emotional work, no animation change).
  - Reaction pills: Use SymbolView icons (NOT emoji). iOS: `heart` + `face.expressionless`. Android: `favorite_border` + `sentiment_neutral`. Never use ❤ or 😐 emoji as UI elements — inconsistent with the project's SymbolView icon system.
  - Selected reaction: `text-accent` on label + icon (same pattern as TagGroup selected item in idle-state). Unselected reaction dims to `foreground/20`.
  - Share button: SymbolView `square.and.arrow.up` (iOS) / `share` (Android), top-right of card. 44px minimum touch target.

- **User journey arc (Design D8):** The emotional arc for the daily quotes screen is: (1) Preference setup → feel seen in the theme choices → (2) First curated quote → "this is for me-ish" → (3) After first session → next day, first session-derived quote with distinct label → "this is actually from ME." The nudge text below the curated quote ("Try a session for a more personal quote") is the bridge between steps 2 and 3. Copy must match Xolace voice: invitation, not upsell.

- **Accessibility & touch targets (Design D9):**
  - Reaction pills: minimum 44px touch target height (use `hitSlop={8}` on PressableFeedback if padding alone is insufficient)
  - Theme tags in PreferenceSetupSheet: `accessibilityRole="checkbox"` on each TagGroup.Item, `accessibilityLabel="[theme name], selected"` / `"[theme name], unselected"` based on selection state
  - Share button: `accessibilityLabel="Share this quote"`, `accessibilityRole="button"`
  - Quote text: `accessibilityRole="text"` or leave default — the label above it ("Your quote today") should be `accessibilityLabel="Your quote today: [quote text]"` so VoiceOver reads the full context
  - Preference setup step indicator: Two dot indicators at bottom of setup card. Active dot = `bg-accent`, inactive = `bg-foreground/20`. `accessibilityLabel="Step 1 of 2"` / `"Step 2 of 2"` on the step container.

- **Widget forward-compatibility (iOS):** Write session-derived quote text to an iOS shared app group container (e.g., `group.com.xolaceincorg.xolace`) in addition to Convex. This PR adds the write only; no widget UI built. Required because iOS WidgetKit cannot make network calls.
- **Widget forward-compatibility (Android):** Android widgets read from SharedPreferences via the app's process. Write the session-derived quote to SharedPreferences in the nightly cron completion callback. Deferred to the widget UI PR — not required in this PR since Android widget development can read from Convex directly during that phase.

## Session-Derived Quote Prompt (draft constraints — test before shipping)

The prompt must produce a quote that:
- Is 1-2 sentences max
- Feels poetic but grounded (not therapy-speak, not motivational poster)
- Can be understood without knowing the user's full context (standalone)
- Is written in second person ("You've been carrying something...") or first person ("There's a weight I'm starting to name.")
- Does NOT reference specific details from the session (privacy: the card may be shared publicly)
- Does NOT contain platitudes ("Everything happens for a reason," "You've got this")
- Passes the "would someone screenshot this?" test

Draft system prompt direction: "You are given the emotional themes from a user's recent reflections. Generate one beautiful, honest sentence (or two short sentences) that captures the emotional experience without being specific. It should feel like something a thoughtful writer found for themselves and wanted to keep. Not advice. Not affirmation. An observation."

  ### Progress Indicator

  import { Canvas, Path, Skia } from "@shopify/react-native-skia";
  import { useEffect, useMemo } from "react";
  import {
    useDerivedValue,
    useSharedValue,
    withTiming,
  } from "react-native-reanimated";
  
  interface CircleProgressProps {
    progress: number; // 0 to 1
    size?: number;
    strokeWidth?: number;
  }
  
  export function CircleProgress({
    progress,
    size = 44,
    strokeWidth = 3,
  }: CircleProgressProps) {
    const radius = (size - strokeWidth) / 2;
    const center = size / 2;
  
    const animatedProgress = useSharedValue(progress);
  
    useEffect(() => {
      animatedProgress.value = withTiming(progress, { duration: 400 });
    }, [progress, animatedProgress]);
  
    const backgroundPath = useMemo(() => {
      const path = Skia.Path.Make();
      path.addCircle(center, center, radius);
      return path;
    }, [center, radius]);
  
    const progressPath = useDerivedValue(() => {
      const path = Skia.Path.Make();
      const startAngle = -90; // Start from top
      const sweepAngle = animatedProgress.value * 360;
  
      path.addArc(
        {
          x: strokeWidth / 2,
          y: strokeWidth / 2,
          width: size - strokeWidth,
          height: size - strokeWidth,
        },
        startAngle,
        sweepAngle
      );
  
      return path;
    }, [animatedProgress, size, strokeWidth]);
  
    return (
      <Canvas style={{ width: size, height: size }}>
        {/* Background circle */}
        <Path
          path={backgroundPath}
          style="stroke"
          strokeWidth={strokeWidth}
          color="rgba(255, 255, 255, 0.15)"
          strokeCap="round"
        />
        {/* Progress arc */}
        <Path
          path={progressPath}
          style="stroke"
          strokeWidth={strokeWidth}
          color="rgba(255, 255, 255, 1)"
          strokeCap="round"
        />
      </Canvas>
    );
  }

usePatternComposer
A React hook for composing and playing a custom Pattern. The pattern is parsed on mount and whenever it changes. Resources are released automatically on unmount.

const { play, parse, isParsed } = usePatternComposer(pattern);

Parameters
pattern (optional)
Type: Pattern

A haptic pattern to parse on mount. When provided, the pattern is parsed automatically and re-parsed whenever the value changes. Resources are released on unmount.

Returns
play()
Plays the parsed pattern.

stop()
Stops the active pattern.

parse(pattern: Pattern)
Parses and replaces the current pattern.

isParsed()
Returns true if a pattern has been parsed and is ready to play.

Example
import { usePatternComposer } from 'react-native-pulsar';
import { Gesture } from 'react-native-gesture-handler';

const pattern = {
  discretePattern: [
    { time: 0, amplitude: 1, frequency: 0.5 },
    { time: 100, amplitude: 0.5, frequency: 0.5 },
  ],
  continuousPattern: {
    amplitude: [
      { time: 0, value: 0 },
      { time: 200, value: 1 },
      { time: 400, value: 0 },
    ],
    frequency: [
      { time: 0, value: 0.3 },
      { time: 400, value: 0.8 },
    ],
  },
};

function MyComponent() {
  const composer = usePatternComposer(pattern);

  const tap = Gesture.Tap().onEnd(() => {
    composer.play();
  });

  return <GestureDetector gesture={tap}>...</GestureDetector>;
}

useRealtimeComposer
A React hook for real-time haptic control. Provides live amplitude and frequency modulation, useful for gesture-driven or continuously evolving haptic experiences. Haptics stop automatically on unmount.

const { set, playDiscrete, stop, isActive } = useRealtimeComposer();

Parameters
useRealtimeComposer doesn’t take any parameters.

Returns
set(amplitude: number, frequency: number)
Updates the ongoing haptic with new amplitude and frequency values.

playDiscrete(amplitude: number, frequency: number)
Plays a single discrete haptic event.

stop()
Stops the active haptic.

isActive()
Returns true if a haptic is currently playing.

Example
import { useRealtimeComposer } from 'react-native-pulsar';
import { Gesture } from 'react-native-gesture-handler';

function MyComponent() {
  const realtime = useRealtimeComposer();

  const pan = Gesture.Pan()
    .onUpdate((e) => {
      const amplitude = Math.min(Math.abs(e.velocityY) / 1000, 1);
      realtime.set(amplitude, 0.5);
    })
    .onEnd(() => {
      realtime.stop();
    });

  return <GestureDetector gesture={pan}>...</GestureDetector>;
}

useAdaptiveHaptics
A React hook that plays haptics from a cross-platform AdaptivePreset. It automatically selects the correct configuration for the current platform (iOS or Android). Each platform can provide either a custom Pattern or a native preset function.

const { play } = useAdaptiveHaptics(preset: AdaptivePreset);

Parameters
preset: AdaptivePreset
An object with ios and android keys. Each value is either a Pattern object or a function that triggers a native preset directly.

type AdaptivePresetConfig = (() => void) | Pattern;

type AdaptivePreset = {
  ios: AdaptivePresetConfig;
  android: AdaptivePresetConfig;
};

Returns
play()
Plays the haptic for the current platform. If the platform config is a function, it is called directly. If it is a Pattern, it is played via the pattern composer.

Example
import { useAdaptiveHaptics, Presets } from 'react-native-pulsar';

const adaptivePreset = {
  ios: Presets.Success,      // native iOS preset function
  android: {                 // custom pattern for Android
    discretePattern: [
      { time: 0, amplitude: 1, frequency: 0.5 },
      { time: 150, amplitude: 0.6, frequency: 0.4 },
    ],
    continuousPattern: {
      amplitude: [],
      frequency: [],
    },
  },
};

function MyComponent() {
  const haptics = useAdaptiveHaptics(adaptivePreset);

  return (
    <Button onPress={haptics.play} title="Tap me" />
  );
}

HapticSupport
The haptic capability level of the current device, returned by Settings.getHapticsSupportLevel().

import { HapticSupport } from 'react-native-pulsar';

enum HapticSupport {
  NO_SUPPORT = 0,
  LIMITED_SUPPORT = 1,
  STANDARD_SUPPORT = 2,
  ADVANCED_SUPPORT = 3,
}

On Android, Settings.forceHapticsSupportLevel(level) uses these exact numeric enum values when selecting the fallback path. If you force a mode while validating custom patterns, make sure you pass the exported HapticSupport enum rather than app-local aliases.

Example
import { Settings, HapticSupport, Presets } from 'react-native-pulsar';

const support = Settings.getHapticsSupportLevel();

if (support >= HapticSupport.STANDARD_SUPPORT) {
  Presets.dogBark();
}

Types
Pattern
Describes a complete haptic pattern with discrete pulses and continuous envelope curves.

type Pattern = {
  discretePattern: Array<{
    time: number; // Milliseconds from pattern start
    amplitude: number; // Intensity (0-1)
    frequency: number; // Sharpness (0-1)
  }>;
  continuousPattern: {
    amplitude: Array<{
      time: number; // Milliseconds from pattern start
      value: number; // Amplitude value (0-1)
    }>;
    frequency: Array<{
      time: number; // Milliseconds from pattern start
      value: number; // Frequency value (0-1)
    }>;
  };
};

Use discretePattern for distinct taps and impacts. Use continuousPattern envelopes to shape a sustained haptic over time.

RealtimeComposerStrategy
Android-only. Controls how useRealtimeComposer simulates continuous haptics, since Android has no native continuous haptic API. Pass one of these values to Settings.setRealtimeComposerStrategy().

import { RealtimeComposerStrategy } from 'react-native-pulsar';

enum RealtimeComposerStrategy {
  ENVELOPE = 0,
  PRIMITIVE_TICK = 1,
  PRIMITIVE_COMPLEX = 2,
  ENVELOPE_WITH_DISCRETE_PRIMITIVES = 3,
}

Value	Description
ENVELOPE	Approximation based on the Envelope API. Allows control over amplitude and frequency, but the signal oscillates and can be unstable. Available on Android API 36+.
PRIMITIVE_TICK	Approximation using the Composition API TICK primitive at varying intervals. Amplitude is controllable; frequency is simulated by the timing between ticks. Signal is discrete rather than continuous.
PRIMITIVE_COMPLEX	Similar to PRIMITIVE_TICK, but uses multiple primitives depending on the requested frequency.
ENVELOPE_WITH_DISCRETE_PRIMITIVES	Default. Hybrid strategy. Uses the Envelope API for continuous events (API 36+) and composition primitives for discrete events (API 33+). Best of both worlds when both event types are used.
Example

import { Settings, RealtimeComposerStrategy } from 'react-native-pulsar';

Settings.setRealtimeComposerStrategy(RealtimeComposerStrategy.PRIMITIVE_COMPLEX);
