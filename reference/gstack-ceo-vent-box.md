## Vision

### 10x Check
Voice Vent isn't a feature — it's a destruction ritual made digital. The paper-burning
users don't want a tool. They want a ceremony: speak your weight into the void, feel
witnessed without judgment, then watch it disappear for real. The 10x version is when
the moment of "Gone." becomes the most emotionally resonant 3 seconds in any app on
the user's phone. Characters scatter like ash. The screen goes dark. The word "Gone."
settles where the fire was. One firm haptic pulse. Silence. That's the campfire moment
made tactile.

7-user research confirms: 5/7 would use the app; 4/7 named voice as the differentiator;
3/7 named the visual destruction as emotionally necessary. This is unusually unambiguous
demand validation for a product at this stage.

## Architecture — Implementation Blueprint

### Navigation Entry Point
The Voice Vent screen is accessed from the Idle state of the reflect screen (`src/features/reflect/components/states/idle-state.tsx`). The existing Timeline Icon button (`<TimelineIcon>`) currently routes to timeline screen for `timelines`. This routing changes, probably icon to changes: tapping the new icon rather opens some kind of menu design(yet to be decided) then tapping on the vent/release/something will now navigates to `voice-vent.tsx` (a stack push with `gestureEnabled: false`, so users cannot swipe back mid-session. but there will be a back or an x button to send the user back to Idle).

### Burn/Keep Mirror Toggle (defer this implementation)
Location: `src/features/reflect/components/states/mirror-state.tsx` at session end (after user confirms "That's it"). A `<Switch>` or `<Toggle>` component labeled "Keep a copy of this reflection?" defaults to OFF. On session completion, the `completeSession` Convex mutation in `convex/sessions.ts` receives `kept: boolean`. If `kept: false` (default), the mutation nulls `rawInput` and `mirrorText` fields on the session row immediately on completion. If `kept: true`, content is retained for timeline. The `kept` field is written on every session completion — non-optional, not a patch-if-present.

### Trust Disclosure Trigger
- Predicate: `ventPrivacyAcknowledged` not set in Zustand persist store (checked in `use-voice-vent.ts` before any mic action)
- Trigger: fires on the first tap of the mic button or any ui we specify on the Voice Vent screen, before `PermissionCheck`
- Dismissal behavior: if user dismisses without accepting, mic does NOT activate; user returns to Idle
- Acceptance: sets `ventPrivacyAcknowledged: true` in Zustand persist; never shown again
- Content: "Your voice is never stored. We send audio to ElevenLabs for a real-time response — nothing is saved on our servers. Anthropic (our AI provider) may retain audio inputs for up to 30 days for safety auditing; we cannot control that side."

### Schema Changes
```typescript
// convex/schema.ts — sessions table addition
// Optional in schema to handle legacy sessions (pre-feature) that were never written.
// All new sessions write kept explicitly on completion (initiateSession does NOT write it;
// completeSession always patches it). Guards treat undefined === false.
kept: v.optional(v.boolean()),

// emotional_profiles additions for circuit breaker
// Reset logic: lazy — checked at vent connection time; reset if lastResetAt < start-of-today-UTC
// No cron needed: checkAndIncrementCap mutation resets counters inline when stale
ventDailyMinutesUsed: v.optional(v.number()),    // accumulated agent-minutes today
ventDailyResetAt: v.optional(v.number()),         // timestamp of last reset (UTC midnight)
```

Convex `kept` field handling: `reflectionDistiller` and `reflectionAnonymizer` guards treat BOTH `kept === false` AND `kept === undefined` as "do not process." Guard: `if (session.kept !== true) return;`

### Circuit Breaker: checkAndIncrementCap
`convex/vent.ts` exports `checkAndIncrementCap` as a public `mutation` (with `requireAuth()` guard, callable via `useMutation` from the client):
- Reads `ventDailyMinutesUsed` and `ventDailyResetAt` from `emotional_profiles`
- If `ventDailyResetAt < start-of-today-UTC`: reset both fields to 0 / now
- If `ventDailyMinutesUsed >= DAILY_CAP` (from `process.env.ELEVENLABS_DAILY_CAP_MINUTES`, default 2): return `{ allowed: false }`
- Otherwise: increment `ventDailyMinutesUsed` by estimated session duration (3 min pessimistic); return `{ allowed: true }`
- Called from `use-vent-circuit-breaker.ts` hook via `useMutation` before agent connects; result is blocking (no optimistic skip)

### Voice Vent State Machine
```
TrustCheck (on first mic tap only)
  → [not acknowledged] → TrustDisclosureSheet
    → [dismissed] → Idle (no mic)
    → [accepted] → PermissionCheck
  → [acknowledged] → PermissionCheck

PermissionCheck
  → [denied] → PermissionDenied → Idle
  → [granted] → CapCheck (calls checkAndIncrementCap)
    → [allowed] → AgentConnecting
      → [connected] → AgentListening (haptic pulse starts: 2s interval, light)
        → [VAD silence 2s / SDK end-of-utterance event] → AgentResponding (haptic stops)
          → [response complete / SDK onMessage event] → Dissolving (fire animation, ~1200ms)
            → [animation peak, ~600ms in] → HapticPulse (Haptics.notificationAsync(NotificationFeedbackType.Success) for android and custom haptic for IOS)
              → [animation end] → IntensityCheck (lighter/same/heavier/unsure)
                → [heavier] → SafetyPrompt (existing escalation UI) → Idle
                → [other] → Idle (clean, session marked completed)
      → [cap exceeded] → FallbackConnecting → FallbackResponding → Dissolving (same path)
      → [ElevenLabs SDK error] → FallbackConnecting → FallbackResponding → Dissolving
      → [network failure] → Error (retry or dismiss to Idle)
    [DEPLOY-TIME CONFIG: if Week-0 spike returns NO-GO, ship Approach B blueprint instead of agent path]
```

VAD source: ElevenLabs Conversational AI SDK handles voice activity detection natively. The client listens to the SDK's `onUserStopSpeaking` or equivalent event (exact event name to be confirmed during the Week-0 spike and documented in `reference/elevenlabs-sdk-validation.md`). No app-side silence timer is implemented.

### Approach B Fallback Blueprint (if Week-0 spike fails)
If `@elevenlabs/react-native` SDK fails validation, Approach B replaces the agent with:
```
AgentListening (expo-speech-recognition active)
  → [user stops / 3s silence] → STT complete
    → [transcript] → Convex internalAction: ventProcessEphemeral

```
Approach B reuses: `expo-speech-recognition`, `use-mirror-audio` (adapted), `convex/ai/process.ts` pattern. Estimated additional effort if needed: human ~1 week / CC ~2 hours.

## Review Findings (S1–S11)

### S1 — Architecture: Safety Classifier Gap (RESOLVED)

ElevenLabs Conversational AI processes audio entirely in their cloud — Xolace servers never see content. `evaluateSafeguard()` requires a text transcript. Without intervention, Approach A has no safety check.

**Resolution**: Parallel STT safety path. `expo-speech-recognition` runs simultaneously with the ElevenLabs agent — for safety classification only, never stored.

```
Mic ─────────────────┬──────────────────────────────────────
                     │                                      │
                     ▼                                      ▼
           ElevenLabs Agent SDK                expo-speech-recognition (STT)
           (affirming response)                (safety classification only)
                     │                                      │
                     ▼                                      ▼
           "Gone." animation             Convex internalAction: classifyVentSafety
                                         → OpenAI moderation + Anthropic classify
                                         → evaluateSafeguard()
                                         → escalation_event written (flag only)
                                         → if crisis: surface resources
```

**Implementation requirements from S1**:
- `classifyVentSafety` internalAction in `convex/vent.ts`: receives transcript, runs moderation + safeguard, writes escalation_event flag only (no transcript stored)
- AppState backgrounding handler in `use-voice-vent.ts`: on app background event, disconnect agent gracefully, skip "Gone." animation, return to Idle
- Circuit breaker race condition: Convex MVCC guarantees serialized reads on `checkAndIncrementCap` — no additional guard needed

### S2 — Error & Rescue Map

| Codepath | What can go wrong | Rescued? | User sees |
|----------|-------------------|----------|-----------|
| ElevenLabs agent connect | Network unreachable | YES → Error/Idle | Error screen |
| ElevenLabs agent connect | API token expired | YES → Error/Idle | Error screen |
| ElevenLabs agent connect | SDK throws unknown error | YES → Error/Idle | Error screen |
| ElevenLabs agent listening | User revokes mic mid-session | YES → Error | "Microphone unavailable" |
| ElevenLabs agent listening | App backgrounded | YES → Idle | Silent exit (AppState) |
| ElevenLabs agent listening | WebSocket drops mid-session | YES → Error/Idle | Error screen |
| STT (parallel safety stream) | STT fails mid-session | YES → skip classify | No safety check (WARN logged) |
| STT (parallel safety stream) | STT timeout | YES → skip classify | No safety check (WARN logged) |
| classifyVentSafety action | Anthropic 429/500 | YES → skip classify | No safety check (WARN logged) |
| classifyVentSafety action | Returns malformed result | YES → skip classify | No safety check (WARN logged) |
| checkAndIncrementCap mutation | DB call fails | Fail open (allow) | Agent proceeds |
| checkAndIncrementCap mutation | Env var not set | Default 2 min cap | Agent proceeds |
| "Gone." animation | Reanimated worklet crash | Skip anim, show text | "Gone." text only |
| postSessionMood write | DB call fails | ⚠️ GAP — silent loss | Mood data lost |

**GAP 1 (minor)**: If `postSessionMood` DB write fails, mood data is silently lost — including the "heavier" path that triggers safety resources. Add retry (max 3 attempts) before giving up.

**GAP 2 (operational)**: When STT fails and safety classification is skipped, log at WARN level with session_id + timestamp so it surfaces in Convex dashboard. Never log transcript content.

**S2 implementation requirement**: `classifyVentSafety` must handle empty/short transcripts — empty string → skip (no risk, no content); <3 words → classify normally; ElevenLabs agent response audio transcript (if captured) → must NOT be classified (only user speech goes to safety).

### S3 — Security (RESOLVED)

ElevenLabs agent SDK requires credentials. Raw API key client-side = key exposure.

**Resolution**: `getVentSessionToken` Convex action generates a signed short-lived session token server-side. Client never holds the API key. Same pattern as existing `convex/ai/tts.ts` which uses `ELEVENLABS_VOICE_API_KEY` server-side only.

PostHog audit result: existing `reflection_submitted` sends only metadata (`entry_type`, `input_length`, `freeze_occurred`) — no content. `captureTouches: true` is safe because only `propsToCapture: ['testID']` elements are captured and vent has no text inputs. Clean.

### S4 — Data Flow & Interaction Edge Cases

| Interaction | Edge case | Handled? | Fix |
|-------------|-----------|----------|-----|
| Tap start button | Double-tap while connecting | Needs fix | `isConnecting` ref blocks 2nd tap |
| Trust disclosure | Accept + immediately background | YES | AppState exits session |
| Agent listening | Complete silence (no speech) | ⚠️ Needs fix | 60s max-session timeout → graceful exit |
| Agent listening | Very long speech (>5 min) | Needs validation | SDK VAD timeout behavior — confirm in Week-0 |
| classifyVentSafety | Empty transcript | YES | Skip — no risk detected |
| Cap exceeded | Show usage limit + support CTA | Needs implementation | UsageLimitSheet state (not FallbackConnecting — see CP3) |
| postSessionMood write | Network drops post-"Gone." | Needs fix | Retry max 3 attempts |

**S4 implementation requirements**:
1. **Silence timeout**: 60-second max session duration. If no `onUserStartSpeaking` event within 60s, gracefully exit to Idle.
2. **Double-tap guard**: Set `isConnecting = true` immediately on first tap; block subsequent taps until `AgentListening` or error state.
3. **postSessionMood retry**: 3 attempts with exponential backoff before silent give-up. Log failure at WARN.

### S7 — Performance: Dual Audio Stream (added to Week-0 spike)

iOS `AVAudioSession` allows only one active audio session at a time. Running `expo-speech-recognition` (STT for safety) in parallel with `@elevenlabs/react-native` (real-time agent) may cause one stream to silence the other.

**Resolution**: Added to Week-0 spike validation scope. If dual streams conflict on iOS:
- Primary fallback: client-side keyword detection (no parallel mic needed)
- Secondary fallback: run STT safety check after agent session ends (transcript reconstructed from ElevenLabs end-of-session event if available, otherwise skip safety)
- Document outcome in `reference/elevenlabs-sdk-validation.md`
